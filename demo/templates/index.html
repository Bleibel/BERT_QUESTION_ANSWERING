Methodology

1

Tokenization

The passage and question are tokenized using WordPiece. Special tokens [CLS] and [SEP] mark boundaries.

2

Sliding Window

Long passages exceeding the model's max length are split into overlapping chunks with a configurable stride to preserve context.

3

Span Prediction

Micro-BERT outputs start/end logits. Softmax converts these to probabilities, and the highest-probability span is selected as the answer.

4

Answer Selection

Across all chunks, the span with the highest confidence score is chosen and mapped back to the original text positions.

About this Project

This project implements an extractive question-answering system for a university NLP course. Instead of using a 340M-parameter BERT-large model, we designed and trained a custom Micro-BERT with only ~10 million parameters.

  • Model: Custom Micro-BERT (4 layers, 256 hidden, 4 heads)
  • Parameters: ~10,120,450 (3% of BERT-large)
  • Dataset: SQuAD 1.1 (Stanford Question Answering Dataset)
  • Metrics: Exact Match (EM) and F1-score
  • Features: Sliding-window chunking, confidence scoring, REST API, training pipeline

The full source code, training scripts, evaluation, and analysis are available in the repository. Run python train.py to fine-tune from scratch, or python run_evaluation.py to test.
